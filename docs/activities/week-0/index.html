Syllabus

Let's take a look at the syllabus =>

Every week there will be activities

The class structure is to talk about new stuff in our joint, two section, very large, class. Then in each section we will do things that will build towards a final project. That project is a completely performed, recorded, and analysed usability test.

Today we will...

  1. Meet our teams and decide our roles.
  2. Choose, or at least discuss, which features will be tested.
  3. Tour the Usability trailer.
  4. Ask any questions you might have.

Meet your teams

You were introduced to your teams the other day. Now is the time to decide which roles to fill. There are many things to be done. Below is a non-exhaustive list. So if you think of anything else, or if we discover something during the semester make sure you add it.

  • Team Lead
  • Test writer
  • Document manager
  • Test subject liasion
  • Consent writer
  • Data analyst
  • Test designer
  • The unknown...

Choose a role, have an election, draw straws, or play rock, paper, scissors, and then let me know who is doing what.

Choose your features...

The usability tests we will be writing will be performed on the features you have developed in the past semester. The criteria for which features to test will be as follows:

  • The features are complete in both CRUD and MVP
  • They are properly styled
  • All error, success, and status messages are implemented.
  • The team agrees

We should be able to test three features by the end of this semester.

Visit the trailer...

We have a mobile usability testing lab that we will be using for this class. We will be taking a tour of it and I will show you how to use it safely. We will be spending most of our time in there.

Questions...

If you have any questions - now would be a good time to ask.

Accessibility basics

See also: https://developer.mozilla.org/en-US/docs/Learn/Accessibility/HTML

  • Semantics: Can you use a button?
  • Content: Can the button describe its function?
  • Focus: If it's not a button, can you trigger it anyway?
  • Tables: scope and colspan
  • Inputs: label for id.
  • Images: alt text
  • Source order: Javascript, but also CSS (i.e. flex-order)
